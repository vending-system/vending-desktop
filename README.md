# 🖥 Vending System Desktop

Десктопное приложение для франчайзера — личный кабинет для управления сетью вендинговых аппаратов. Разработан как нативный desktop-клиент (без WebView) на базе Avalonia UI.

## 🛠 Технологии

- **Avalonia UI** — кроссплатформенный UI-фреймворк (.NET)
- **MVVM** — архитектурный паттерн
- **CommunityToolkit.Mvvm** — реализация MVVM
- **LiveCharts2** — графики и диаграммы
- **JWT** — авторизация через API

## 🚀 Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/vending-system/vending-desktop.git
cd vending-desktop
```

### 2. Настроить URL API

В `Services/ApiService.cs` укажи адрес запущенного API:

```csharp
private const string BaseUrl = "https://localhost:5001/api";
```

### 3. Запустить

```bash
dotnet run
```

## 📋 Функциональность

### 🔐 Авторизация
- Вход по логину и паролю с JWT
- Отображение данных сотрудника (фото, ФИО, роль)
- Dropdown с профилем пользователя

### 🏠 Главная страница
- **Эффективность сети** — процент работающих аппаратов
- **Состояние сети** — диаграмма по статусам (работает / не работает / на обслуживании)
- **Сводка** — данные по продажам, инкассации и обслуживанию
- **Динамика продаж за 10 дней** — график с фильтром по сумме / количеству
- **Новости** — лента новостей франчайзера
- Поддержка перетаскивания и скрытия блоков

### 📊 Монитор ТА
- Таблица всех аппаратов в реальном времени
- Колонки: №, ТА, Связь, Загрузка, Деньги, События, Оборудование, Информация
- Фильтры: по общему состоянию, типу подключения, дополнительным статусам
- Кнопка «Применить» для применения фильтров
- Поле «Итого аппаратов» и «Денег в аппаратах» с пересчётом по фильтру

### ⚙️ Администрирование

**Торговые автоматы:**
- Переключение между режимами отображения: таблица / плитка
- Фильтр по названию, пагинация, выбор количества строк
- Группировка по франчайзи
- Экспорт в CSV, PDF, HTML
- Добавление, редактирование, удаление ТА
- Отвязка модема от ТА

**Компании, Пользователи, Модемы** — управление справочниками

### 🔔 Push-уведомления
- **Критические** (красный, 10 сек) — нет сдачи, замятие, перегрев
- **Предупреждения** (жёлтый, 7 сек) — низкий запас, необходимость обслуживания
- **Информационные** (зелёный/синий, 5 сек) — успешная оплата, выдача товара
- Toast-уведомления с кнопкой закрытия
- Очередь с приоритетами: Критические > Предупреждения > Информационные
- Логирование всех уведомлений

## 🔗 Интеграция с API

Приложение работает через REST API. Для эмуляции realtime-данных (статус связи, загрузка товаров, денежные средства) используется эндпоинт `/api/machines/realtime`.
